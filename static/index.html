Personas

A persona is an AI identity you can switch into per chat — like "code reviewer", "journal therapist", or "drift writer". Each persona's identity text replaces Layer 0 (below) when active. Switch via the topbar; the default persona uses the Layer 0 identity edited below.

Identity (Layer 0)

A persistent system prompt loaded into every conversation, no matter the wing. Stored in ~/.mempalace/identity.txt. Describe who the AI is and who you are — not what you're working on. Project / topic context belongs in the per-wing system prompt below.

Wings & rooms — how to think about them

            
    
              
  • Wing = a top-level domain. A project, a person, or a big topic. Examples: personal, dos-clone, health, work.
    • 
              
  • Room = a facet inside a wing. For a project: auth, gameplay, billing. For a person: conversations, preferences.
    • 
              
  • Auto-extracted facts land in rooms named facts_<type> (e.g. facts_preference) inside the active wing.
    • 
              
  • Memory recall is scoped to the current wing so unrelated context can't bleed in.
