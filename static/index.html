Wings & rooms — how to think about them

            
    
              
  • Wing = a top-level domain. A project, a person, or a big topic. Examples: personal, dos-clone, health, work.
    • 
              
  • Room = a facet inside a wing. For a project: auth, gameplay, billing. For a person: conversations, preferences.
    • 
              
  • Auto-extracted facts land in rooms named facts_<type> (e.g. facts_preference) inside the active wing.
    • 
              
  • Memory recall is scoped to the current wing so unrelated context can't bleed in.
